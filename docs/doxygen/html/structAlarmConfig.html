Detailed Description

In-memory representation of one alarm entry.

Alarm records are persisted as a compact pipe-separated list in NVS under the alarm/alarms key. Each record stores the fields in this order: enabled,hour,minute,sound,schedule,title.

The title field is percent-encoded before storage so commas and pipe characters cannot corrupt parsing.
