Detailed Description

HTTP handlers for the LED clock single-page web interface.

The web layer shares the same compact alarm record format as the persistence layer in main.cpp. Titles are percent-decoded here so the browser can send them safely inside a comma- and pipe-delimited payload.

◆ handleApiAlarms()

void handleApiAlarms ( )

Saves the full alarm list received from the browser.

Validates and stores the full alarm list submitted by the web UI.

The browser submits alarms in one compact form field: enabled,hour,minute,sound,schedule,title|...

Titles are percent-encoded so commas and pipes remain reserved as field and record delimiters. The parser requires the full six-field record layout and skips invalid records rather than accepting partially corrupt data.

Returns the current device state, including alarms, as JSON.

Returns the current device status and alarm list as JSON.

Alarm objects are expanded into named JSON fields here even though they are stored compactly in NVS. This keeps the browser code straightforward while allowing the firmware to retain its compact persistence format.
